Foundations

Foundations

Os blocos atómicos do design da PCDIGA: cores, tipografia, espaçamento e tudo o que define a linguagem visual antes de chegar a componentes. Tokens oficiais sincronizados com o Figma.

Light + Dark mode A PCDIGA suporta os dois modes. Quase todos os tokens têm valores diferentes em light e dark. Usa o botão no topo direito para alternar — toda a documentação reage ao mode ativo.

Usage

✓ Faz

Usar var(--brand-orange) em vez de #FF4D23. Tokens funcionam automaticamente em light e dark.

✗ Evita

Hardcodar valores hex. Vais ter que duplicar tudo em dark mode e a manutenção fica impossível.

✓ Faz

Usar --stock-available apenas para "Em stock" — verde tem semântica clara para o utilizador.

✗ Evita

Verde para descontos ou preços. Cor de desconto é amarelo (#F5CC45) ou laranja brand.

✓ Faz

Um sistema por contexto: PCDIGA (laranja) para B2C, Empresarial (azul) para B2B. Cada contexto define a sua cor primária.

✗ Evita

Misturar laranja e azul como cores principais na mesma página. O utilizador deixa de saber em que sistema está.

Acessibilidade

Contraste WCAG AA Todos os tokens de marca foram testados para AA. Stock #FF0000 em light tem rácio limítrofe — para textos pequenos preferir o tom dark #FF6B6B com fundo branco também passa AA.

Pesos disponíveis

Lato tem 5 pesos em Google Fonts (100, 300, 400, 700, 900). A PCDIGA usa apenas três para manter consistência:

Regular
400 · body, descrições, leitura
Bold
700 · h1–h3, CTAs, ênfase, labels
Black
900 · só Display / hero
Regra prática Apenas três pesos para toda a UI. Não usar 100 ou 300 (light) — comprometem legibilidade em produto e diluem a identidade visual.

Regras práticas

✓ Faz

Line-height generoso (1.5–1.6) em texto corrido para legibilidade.

✗ Evita

Letter-spacing negativo em texto pequeno. Só em títulos grandes (h1, display).

✓ Faz

Limitar largura de parágrafos a ~60–70 caracteres.

✗ Evita

All-caps em parágrafos. Reservar para badges, labels muito curtos ou eyebrows.

Motion

Motion

Movimento serve a UX — comunica feedback, hierarquia e continuidade. Nunca decoração.

Durações

TokenValorQuando usar
--duration-fast120msHover, focus, micro-interações
--duration-base200msToggle, dropdowns, accordions
--duration-slow320msModals, page transitions

Princípios

✓ Faz

Animar opacidade e transform (rápido, performático).

✗ Evita

Animar width/height/top/left (causa reflow).

✓ Faz

Respeitar prefers-reduced-motion.

✗ Evita

Animações longas em ações frequentes.
