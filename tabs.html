Componente

Tabs

Separadores horizontais para alternar entre vistas relacionadas. Crítico em PDP (Especificações, Avaliações, Info adicional) e área de cliente (Encomendas, Devoluções, Faturas).

<div class="tabs" role="tablist">
  <button class="tabs__item is-active" role="tab" aria-selected="true">
    Especificações
  </button>
  <button class="tabs__item" role="tab" aria-selected="false">
    Avaliações
  </button>
  <button class="tabs__item" role="tab" aria-selected="false">
    Info adicional
  </button>
</div>
Anatomia

Anatomia

Container .tabs com border-bottom + items .tabs__item. Tab ativo herda border-bottom laranja brand. Painel de conteúdo abaixo com .tabs__panel.

Variantes

Variantes

Default

Conteúdo do tab ativo aparece aqui.

<div class="tabs">...</div>
<div class="tabs__panel">
  <!-- conteúdo do tab ativo -->
</div>

Com badge contador

Para indicar quantos items em cada secção. Útil em área de cliente.

<button class="tabs__item is-active">
  Encomendas
  <span class="ds-badge ds-badge--sm">3</span>
</button>

Com ícone

<button class="tabs__item is-active">
  <svg ...></svg>
  Especificações
</button>
States

States

Default + Active + Disabled

Usage

Do's & Don'ts

✓ Faz

3-5 tabs. Mais que isso, considera filtros ou navegação separada.

✗ Evita

8+ tabs com scroll horizontal. Utilizador perde-se.

✓ Faz

Tab default = mais relevante para o utilizador (ex: "Especificações" em PDP).

✗ Evita

Tab default vazio. Se não tem conteúdo, esconde a tab.

✓ Faz

Labels curtos e claros: "Specs", "Reviews", "Info". Verbo+objeto desnecessário.

✗ Evita

"Ver as especificações técnicas" — verboso, ocupa espaço.

Acessibilidade

Acessibilidade

Rolesrole="tablist" no container, role="tab" nos items, role="tabpanel" no painel.
Estadoaria-selected="true" no tab ativo, "false" nos outros.
Associaçãoaria-controls="panel-id" no tab + id correspondente no painel.
Keyboard← → entre tabs, Tab para sair, Enter/Space para ativar (se não auto-ativar com setas).
Disableddisabled + aria-disabled="true".
