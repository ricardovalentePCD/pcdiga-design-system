Componente

Input

Campos de formulário onde o utilizador escreve. O coração de qualquer checkout, registo ou pesquisa. Bons inputs reduzem fricção, maus inputs perdem clientes.

Vai receber a confirmação de encomenda neste endereço.
Anatomia

Anatomia

Um input completo tem 4 partes. Apenas o input é obrigatório — o resto adiciona contexto e clareza.

ParteObrigatórioFunção
input-field__labelRecomendadoDiz ao utilizador o que preencher. Sempre acima do campo.
inputSimO campo onde se escreve.
input-field__helperOpcionalTexto de apoio: formato esperado, motivo do campo.
input-field__errorEm erroSubstitui o helper quando há validação falhada.
States

States

Cada estado comunica algo diferente ao utilizador. Garantir que o feedback visual é imediato e claro.

Default

<div class="input-field">
  <label class="input-field__label" for="name">Nome completo</label>
  <input class="input" id="name" type="text"
         placeholder="O teu nome completo" />
</div>

Focus

Borda laranja + outline subtil. Indica que o teclado está a apontar para este campo. Aplicado automaticamente no :focus.

/* CSS — aplicado automaticamente */
.input:focus {
  border-color: var(--brand-orange);
  box-shadow: 0 0 0 3px rgba(255, 77, 35, 0.15);
}

Filled

Com valor preenchido. Sem estado especial — o conteúdo fala por si.

<div class="input-field">
  <label class="input-field__label" for="email">Email</label>
  <input class="input" id="email" type="email"
         value="joao.silva@gmail.com" />
</div>

Error

Borda vermelha + mensagem de erro específica. Nunca dizer apenas "Erro" — explicar o que está mal.

Falta o símbolo @ no email. 
<div class="input-field">
  <label class="input-field__label" for="email">Email</label>
  <input class="input input--error" id="email" type="email"
         aria-invalid="true" aria-describedby="email-err" />
  <span class="input-field__error" id="email-err">
    Falta o símbolo @ no email.
  </span>
</div>

Disabled

Campo não editável. Cor mais clara, cursor not-allowed. Usar quando o campo é dependente de outro ainda não preenchido.

<div class="input-field">
  <label class="input-field__label" for="dist">Distrito</label>
  <input class="input" id="dist" type="text"
         placeholder="Selecciona o país primeiro" disabled />
</div>
Sizes

Sizes

Três tamanhos. md é o default. Usar sm em filtros e tabelas, lg em hero forms (newsletter, pesquisa principal).

<input class="input input--sm" type="text" />
<input class="input" type="text" />
<input class="input input--lg" type="text" />
TokenPaddingFont sizeQuando usar
input--sm6px 10px13pxFiltros, tabelas, ações inline
input10px 14px14pxDefault — formulários gerais
input--lg14px 16px16pxNewsletter hero, pesquisa principal
Variantes

Variantes

Com helper text

Texto auxiliar que explica o que preencher ou o formato esperado.

9 dígitos. Aparecerá na fatura. 
<div class="input-field">
  <label class="input-field__label" for="nif">NIF</label>
  <input class="input" id="nif" type="text" maxlength="9"
         aria-describedby="nif-h" />
  <span class="input-field__helper" id="nif-h">
    9 dígitos. Aparecerá na fatura.
  </span>
</div>

Required (obrigatório)

Asterisco vermelho ao lado do label indica obrigatoriedade.

<div class="input-field">
  <label class="input-field__label" for="email">
    Email <span class="required">*</span>
  </label>
  <input class="input" id="email" type="email"
         required aria-required="true" />
</div>

Com ícone (input-group)

Usar .input-group para integrar ícones (pesquisa, password reveal, etc.).

<div class="input-field">
  <label class="input-field__label" for="search">Pesquisar</label>
  <div class="input-group">
    <span class="input-group__icon">
      <svg ...></svg>
    </span>
    <input class="input" id="search" type="search" />
  </div>
</div>

Email

<input class="input" type="email"
       placeholder="nome@gmail.com"
       autocomplete="email" />

Telefone

<input class="input" type="tel"
       placeholder="912 345 678"
       autocomplete="tel" />

Password

<input class="input" type="password"
       placeholder="••••••••" />

Number (quantidade)

<input class="input" type="number"
       min="1" max="99" value="1" />
Usage

Do's & Don'ts

✓ Faz

Apenas os campos essenciais. Cada campo extra reduz a probabilidade de o utilizador completar o form.

✗ Evita

"Para o caso de precisarmos um dia". Se não usas, não pedes.

✓ Faz

Label sempre visível, acima do campo. Mantém-se quando o utilizador começa a escrever.

✗ Evita

Placeholder a fazer de label. Desaparece ao escrever — utilizador esquece o que está a preencher.

✓ Faz

Mensagens de erro específicas: "Falta o @ no email", "Password tem que ter 8+ caracteres".

✗ Evita

"Erro", "Inválido", "Tente novamente". O utilizador não sabe o que corrigir.

✓ Faz

Validar quando faz sentido — em onBlur ou submit. Validar em tempo real ao escrever frustra.

✗ Evita

Ficar vermelho ao 1º caracter. O utilizador ainda não acabou de escrever.

Acessibilidade

Acessibilidade

Label associadoSempre <label for="id"> ligado ao id do input. Crítico para screen readers.
RequiredAtributo required + asterisco visual + aria-required="true".
Errosaria-invalid="true" + aria-describedby apontando ao id da mensagem.
AutocompleteUsar autocomplete="email", autocomplete="tel", etc. Browser preenche automaticamente.
Tipo correctotype="email", type="tel", type="number". Mobile adapta o teclado.
Tab orderOrdem natural de cima para baixo. Não usar tabindex com números > 0.
Touch targetMínimo 44px de altura no tap. md e lg garantem; sm só em desktop.
