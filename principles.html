Princípios

Princípios

Cinco princípios que guiam todas as decisões de design na PCDIGA — do botão ao checkout. Não são regras técnicas. São a forma como pensamos antes de desenhar qualquer coisa.

Como usar Sempre que estiveres com uma decisão difícil — qual variante, quantos campos, que copy — volta aqui. Os princípios não dão a resposta exata, mas eliminam as opções erradas.
Princípio 1

Clareza primeiro

O utilizador percebe em segundos o que é, porque importa, e o que fazer a seguir. Se demora a perceber, falhámos.

O que isto significa na prática

Cada página tem um objetivo único. Cada secção contribui para esse objetivo. Cada elemento tem que justificar a sua presença visual.

Se um designer da PCDIGA olha para uma página nossa pela primeira vez e tem de perguntar "o que é isto?" — está mal.

✓ Faz

Headline que explica o que é a página em ≤7 palavras. Subtítulo que reforça em ≤15.

✗ Evita

Headlines vagas, abstratas, ou que precisam de contexto extra para fazer sentido.

✓ Faz

CTA principal visível sem scroll, com texto que descreve a ação ("Configurar PC", "Adicionar ao carrinho").

✗ Evita

"Saber mais" sem contexto, "Clica aqui", ou CTAs que escondem a verdadeira ação.

Princípio 2

Menos é melhor

Cada elemento justifica o seu lugar. Reduzir fricção é mais valioso do que adicionar features. Se há dúvida, retira.

O que isto significa na prática

Configuradores complexos? Uma decisão por step. Forms longos? Apenas os campos essenciais. Páginas de produto? Foco no que decide a compra, não em tudo o que se sabe sobre o produto.

O caminho do "menos" é mais difícil. Adicionar é fácil — qualquer um adiciona. Remover requer perceber o que importa de verdade.

✓ Faz

Antes de adicionar um campo a um form, pergunta: "se este campo não existir, o utilizador consegue avançar?" Se sim, remove.

✗ Evita

Adicionar todas as opções "para quem precisar". Quem precisa raramente compensa o ruído para os outros 95%.

✓ Faz

Mostrar 4-6 produtos em destaque por secção. Suficiente para escolher, sem paralisar.

✗ Evita

12+ produtos em carrosséis horizontais infinitos. Ninguém faz scroll até ao fim.

Princípio 3

Hierarquia clara

Tudo não pode ter o mesmo peso visual. O olho precisa de saber para onde ir. Se tudo grita, nada se ouve.

O que isto significa na prática

Em cada vista, identifica o que é mais importante e dá-lhe peso visual proporcional. O resto suporta — não compete.

Hierarquia faz-se com tamanho, cor, posição, densidade e espaço. Não precisa de ser óbvia para o designer — precisa de ser óbvia para quem está a usar.

✓ Faz

Um (1) CTA primário por vista. Tudo o resto é secundário ou ghost.

✗ Evita

3 botões laranja com o mesmo peso. O utilizador hesita, e quem hesita não converte.

✓ Faz

Preço atual em destaque, preço anterior em secundário, % desconto em pill.

✗ Evita

Preço, badge, stock, parcelas — todos com o mesmo tamanho e peso. Resultado: nenhum se destaca.

Princípio 4

Consistência

Mesmo padrão, mesma estrutura, em todas as páginas. O utilizador aprende uma vez, aplica em todo o lado. Esta é a razão de existir do design system.

O que isto significa na prática

Antes de criar uma variante nova, pergunta: "já existe um padrão para isto?" Se sim, usa. Se não tem como usar, melhora o padrão existente — não inventes paralelo.

Consistência ≠ uniformidade. Significa que diferenças visuais comunicam diferenças reais (estado, prioridade, função) — não acidentes de quem desenhou.

✓ Faz

Usar tokens (--brand-primary, --space-4) em vez de valores hardcoded. Se mudar, muda em todo o lado.

✗ Evita

"Para esta página fica melhor mais escuro" — esta exceção depois multiplica-se em 50 outras.

✓ Faz

Mesma posição para CTAs principais, mesmo padrão para preços, mesma forma de mostrar stock.

✗ Evita

Reinventar o card de produto em cada listagem. O utilizador deixa de reconhecer o que é clicável.

Princípio 5

Servir o objetivo

Bonito não chega. O design existe para guiar o utilizador a uma ação — comprar, configurar, decidir, comparar. Se o design é elogiado mas a conversão não cresce, falhámos.

O que isto significa na prática

Cada página tem um objetivo de negócio. Cada decisão de design pergunta: "isto aproxima ou afasta esse objetivo?" Decoração que não serve o objetivo é ruído.

Isto não significa que o design fica feio. Significa que beleza e função andam juntas — quando colidem, função ganha.

✓ Faz

Antes de cada decisão, formula: "qual o objetivo desta página? esta opção aproxima-me dele?"

✗ Evita

Animações longas, hovers complexos ou efeitos visuais que distraem do CTA.

✓ Faz

Medir. Olhar para os dados. Iterar com base no que funciona, não no que parece bem em screenshot.

✗ Evita

Validar design só por gosto interno. O cliente é quem decide se funciona — através do comportamento dele.

Aplicar

Como aplicar os princípios

Os princípios não são frases bonitas para por num poster. São perguntas para fazer antes, durante e depois de cada decisão.

Antes de começar

Antes de abrir o Figma ou o editor, responde:

  • Qual é o objetivo desta página/feature/componente?
  • Quem vai usar isto, em que contexto?
  • Qual é a ação principal que queremos que aconteça?
  • O que pode bloquear ou confundir essa ação?

Durante o design

Em cada decisão, passa pelos 5 princípios:

PerguntaPrincípio
Isto é claro à primeira vista?1. Clareza
Posso simplificar ou retirar algo?2. Menos
Está claro o que é mais importante?3. Hierarquia
Estou a usar padrões e tokens existentes?4. Consistência
Isto serve o objetivo da página?5. Objetivo

Antes de entregar

Faz o teste dos 3 segundos: pede a alguém que não esteve envolvido para olhar para a página durante 3 segundos. Depois pergunta:

  • O que é esta página?
  • Qual é a coisa mais importante aqui?
  • O que é suposto eu fazer a seguir?

Se a resposta a qualquer destas perguntas for hesitante, volta atrás. Reduz, simplifica, hierarquiza.
