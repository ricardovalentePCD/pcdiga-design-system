Fase 3 · Em planeamento

Padrões

Padrões de e-commerce que combinam componentes para resolver problemas concretos da PCDIGA: comprar, configurar, comparar, fazer checkout.

Padrões críticos PCDIGA

A diferença entre componentes e padrões é importante: um componente é uma peça (botão), um padrão é uma combinação que resolve um problema (página de produto inteira).

PDP — Página de produto

Galeria, preço, variantes, stock, CTA principal, specs, bundle, related.

PLP — Listings

Grid de produtos, filtros laterais, ordenação, paginação, contadores.

Filtros & Sort

Filtros desktop e mobile, ordens disponíveis, filtros aplicados visíveis.

Cart

Mini-cart e cart full. Edição de quantidades, sugestões cross-sell, totais.

Checkout

Steps, validação, métodos de envio, métodos de pagamento, resumo.

Search

Search bar, sugestões, results page, sem resultados.

Empty states

Carrinho vazio, sem favoritos, sem encomendas, sem resultados.

Error states

404, 500, erros de checkout, problemas de stock, falha de pagamento.
