Componente

Button

Os botões representam ações que o utilizador pode tomar — comprar, configurar, adicionar, confirmar. São o ponto de conversão de qualquer página.

Variantes

Variantes

Quatro variantes que comunicam hierarquia de ação. Apenas uma Primary por vista — a ação principal.

Primary

Para a ação principal de cada página. Laranja PCDIGA. Captura a atenção.

<button class="btn btn--primary">Comprar agora</button>

Secondary

Para ações alternativas mas válidas. Border, sem fill. Não compete visualmente com a Primary.

<button class="btn btn--secondary">Ver detalhes</button>

Ghost

Para ações terciárias. Sem fill nem border. Usar quando há muitas ações próximas e não queremos competir.

<button class="btn btn--ghost">Cancelar</button>

Com ícone

Ícone à esquerda reforça a ação. Tamanho 16px, herda currentColor.

<button class="btn btn--primary">
  <svg width="16" height="16" viewBox="0 0 24 24" ...>...</svg>
  Adicionar ao carrinho
</button>
Sizes

Sizes

Três tamanhos. md é o default — usar nos restantes apenas com motivo.

<button class="btn btn--primary btn--sm">Small</button>
<button class="btn btn--primary">Medium</button>
<button class="btn btn--primary btn--lg">Large</button>

States

Cada variante tem todos estes estados. O sistema garante feedback visual consistente.

Focus

Outline de 3px laranja claro à volta. Não remover — é crítico para acessibilidade keyboard. Aplicado automaticamente via :focus-visible.

/* CSS — aplicado automaticamente */
.btn:focus-visible {
  outline: 3px solid rgba(255, 77, 35, 0.4);
  outline-offset: 2px;
}

Disabled

Atributo disabled — não-focável, não-clicável.

<button class="btn btn--primary" disabled aria-disabled="true">
  Indisponível
</button>

Loading

Substituir o texto por um spinner durante operações async (ex: submeter form, processar pagamento). O botão fica não-clicável.

<button class="btn btn--primary btn--loading" aria-busy="true">
  Comprar
</button>

Como link

Para CTAs que navegam (ex: "Ver produto"), usar <a> em vez de <button>.

Ver produto 
<a href="/pdp/123" class="btn btn--primary">Ver produto</a>
Anatomy

Estrutura

Um botão é um elemento <button> ou <a> com a classe .btn + uma variante (--primary, --secondary, --ghost).

Usage

Do's & Don'ts

✓ Faz

Uma só Primary por vista. A Primary é a ação principal — e principal só há uma.

✗ Evita

3 botões laranja a competir entre si. O utilizador hesita e quem hesita não converte.

✓ Faz

Texto verbo + objeto: "Adicionar ao carrinho", "Configurar PC", "Pedir orçamento".

✗ Evita

"Saber mais", "Clica aqui", "OK". Vagos e sem contexto. Diz o que vai acontecer.

✓ Faz

Loading state em ações async (compra, submit). Evita duplo-clique e dá feedback.

✗ Evita

Botão "morto" durante loading sem indicador. O utilizador pensa que falhou e clica de novo.

✓ Faz

Manter os botões alinhados à direita em diálogos, com Primary à direita: [Cancelar] [Confirmar].

✗ Evita

Inverter Primary/Secondary entre diálogos. Quebra a previsibilidade.

Acessibilidade

Acessibilidade

RoleUsar sempre <button>. Para links visuais como botão, usar <a> com a classe.
KeyboardTab para focar. Enter ou Space para ativar.
Focus visibleOutline 3px laranja claro. Visível em qualquer tema.
DisabledAtributo disabled + aria-disabled="true". Não focável.
LoadingAdicionar aria-busy="true". Manter o texto de fundo para screen readers (ainda que invisível).
Icon-onlyObrigatório aria-label="...". Sem isto o screen reader não anuncia nada.
ContrastePrimary 4.5:1 (laranja sobre branco), Secondary herda do tema. AA garantido.
